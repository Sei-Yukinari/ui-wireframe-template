## 5. モーション & アニメーション

### 基本方針
アニメーションは控えめに、操作を妨げない自然な速度で実装する。ユーザーの注意を導き、不要な動きを避ける。

### イージング
| 用途 | 関数 |
|------|------|
| 標準的な遷移 | ease-in-out |
| 登場 | ease-out |
| 退場 | ease-in |

### デュレーション
| 種類 | 時間 | 用途 |
|------|------|------|
| Fast | 150ms | ホバー、フォーカス |
| Normal | 300ms | 一般的なトランジション |
| Slow | 500ms | ページ遷移、モーダル |

### アクセシビリティ
- prefers-reduced-motion 対応：[x] 対応する / [ ] 対応しない

## 7. 命名規則 & 設計ルール

### CSS / クラス命名規則
Utility-first（Tailwind CSS）をベースにしたユーティリティクラスを採用。  
必要に応じてコンポーネント固有の名前空間を使用。

### コンポーネント命名規則
PascalCase をコンポーネント名に使用、props や内部変数は camelCase を使用。
