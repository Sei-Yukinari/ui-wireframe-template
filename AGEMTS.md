# AGENTS.md

## 現在のフェーズ

本プロジェクトは「ワイヤーフレーム設計フェーズ」です。

目的はUIの画面構造とレイアウトの確定のみです。
ビジネスロジックや実装の完成度は求めていません。

---

## 絶対ルール

以下を厳守してください。

- デザインを作り込まない
- 色を付けない（gray系のみ使用可）
- アニメーションを入れない
- アイコンを使わない
- 外部UIライブラリを使わない
- 状態管理を入れない
- API通信を書かない
- Server Actionsを使わない
- データベース処理を書かない
- バリデーションを書かない
- useEffectなど副作用処理を書かない

---

## 技術スタック

- TypeScript
- Next.js（App Router）
- TailwindCSS

### ディレクトリ構造

```
src/
├── app/                      # Next.js App Router
│   ├── layout.tsx            # ルートレイアウト
│   ├── page.tsx              # ホームページ
│   ├── loading.tsx           # ローディングUI
│   ├── error.tsx             # エラーUI
│   ├── not-found.tsx         # 404ページ
│   ├── [dynamic]/            # 動的ルート
│   │   └── page.tsx
│   └── api/                  # API Routes
│       └── [endpoint]/
│           └── route.ts
├── components/               # Reactコンポーネント
│   ├── ui/                   # 汎用UIコンポーネント（Button, Card等）
│   ├── features/             # 機能単位のコンポーネント（StoryList, CommentSection等）
│   └── layouts/              # レイアウトコンポーネント（Header,Footer等）
```

これ以外は使用しない。

---

## ワイヤーフレーム表現ルール

UIは以下のルールで構築する。

- border
- bg-gray-100 / bg-gray-200
- p-_, m-_ による余白
- flex / grid によるレイアウト

見た目の美しさではなく「構造の可視化」を最優先とする。

---

## 出力ルール

- 常にフルファイルを出力する
- 省略しない
- 説明文を出力しない
- コードのみ出力する
- 必ず実行可能なコードにする

---

## ダミーデータについて

- 静的な配列のみ使用する
- 仮データで十分
- 永続化は不要

---

## 画面構造優先事項

優先順位は以下。

1. ページ階層の整理
2. レイアウトブロックの明確化
3. コンポーネント分割
4. UI構造の一貫性

---

## 禁止事項（重要）

以下を生成した場合は失敗とみなす。

- 本番デザイン風UI
- 色付きボタン
- モーダルアニメーション
- 状態管理ロジック
- 実際のCRUD処理
- 外部依存追加

---

## このフェーズのゴール

- 画面一覧をブラウザで確認できる
  - localhostで全画面確認できる
- 各ページのレイアウト構造が明確
- コンポーネント分割の方向性が見える
