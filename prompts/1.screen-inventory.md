# Wireframe生成プロンプト（確定構造版）

## 目的

確定済みScreen Inventoryに基づき、
TODOアプリのワイヤーフレームを生成する。

このフェーズの目的は
「UIレイアウト構造の確定」であり、
ビジネスロジックの実装ではない。

---

## 技術スタック（固定）

- Next.js（App Router）
- TypeScript
- TailwindCSS

外部ライブラリ禁止。

---

## 生成対象ルーティング（厳守）
src/app/
├─ layout.tsx
├─ page.tsx                （ログイン）
├─ dashboard/
│   ├─ layout.tsx
│   └─ page.tsx
├─ task/
│   └─ [id]/
│        └─ page.tsx
└─ settings/
└─ page.tsx

src/components/
├─ Header.tsx
├─ Sidebar.tsx
├─ TaskList.tsx
├─ TaskItem.tsx
├─ BackButton.tsx
└─ CreateTaskModal.tsx

この構造以外のファイルを作成しない。

---

## 画面別レイアウト仕様

### 1. `/`（ログイン）

- 画面中央配置
- Cardコンテナ
- Email入力
- Password入力
- Loginボタン

---

### 2. `/dashboard`

レイアウト：

- Header（上部固定）
- Sidebar（左固定）
- Mainエリア（右）

Main内：
- TaskList表示
- 右下にAdd Taskボタン

---

### 3. `/task/[id]`

Main構造：

- BackButton
- タスクタイトル
- タスク説明
- ステータス表示エリア

---

### 4. `/settings`

- タイトル表示
- ユーザー情報表示エリア
- ログアウトボタン

---

## 画面遷移
- ログイン成功 → /dashboard
- TaskItemクリック → /task/[id]
- BackButtonクリック → /dashboard
- Add Taskクリック → CreateTaskModal表示
- ログアウトクリック → /（ログイン）
- TaskList内のタスクは静的ダミー配列から生成

## モーダル仕様

CreateTaskModal：

- タイトル入力
- 説明入力
- 保存ボタン
- キャンセルボタン

表示ロジックは書かない。
常に表示されていてもよい（Wireframe目的）。

---

## UI表現ルール（厳守）

使用可能：

- border
- bg-gray-100 / bg-gray-200
- flex
- grid
- p-*, m-*
- text-sm / text-base

禁止：

- カラーデザイン
- shadow多用
- rounded過剰使用
- アニメーション
- icon
- 状態管理
- useState
- useEffect
- API通信
- Server Actions
- データ永続化

---

## データ

- 静的ダミー配列のみ使用
- TaskList内で定義
- 3〜5件で十分

---

## 出力ルール（絶対）

- フルディレクトリ構造を提示
- すべてのファイルを完全コードで出力
- 省略禁止
- 説明文禁止
- 実行可能コードのみ

---

## 成功条件

- localhostで全画面遷移確認可能
- レイアウト構造が明確
- コンポーネント分割が合理的
- 実装ロジックは存在しない
- 画面遷移が正しく機能する（useState等の実装は不要）
