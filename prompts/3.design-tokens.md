# UIトークン定義プロンプト

## 目的

TODOアプリのUIをDesign System化するため、
UIトークンを定義する。

将来的なブランド変更・ダークモード対応を前提とした、
拡張可能なトークン設計とする。

---

## 前提

- Next.js
- TypeScript
- TailwindCSS
- SaaS風ミニマルUI
- shadcn/ui導入を想定

---

## 出力形式

トークン定義のみ出力する。
実装コードは出力しない。

# 2. Spacing Tokens

基準単位: 4px

- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- 2xl: 48px

Tailwind基準:
p-1 = xs
p-2 = sm
p-4 = md
p-6 = lg
p-8 = xl

---

# 3. Radius Tokens

- radius-sm: 4px
- radius-md: 8px
- radius-lg: 12px
- radius-xl: 16px

標準使用:
Card → radius-lg
Button → radius-md
Input → radius-md

---

# 4. Shadow Tokens

- shadow-sm:
  0 1px 2px rgba(0,0,0,0.05)

- shadow-md:
  0 4px 6px rgba(0,0,0,0.08)

- shadow-lg:
  0 10px 15px rgba(0,0,0,0.12)

原則:
通常カード → shadow-sm
モーダル → shadow-lg

# 6. Component Variant Rules

## Button Variants

Primary:

- bg-primary-500
- text-white
- hover:bg-primary-600

Secondary:

- bg-surface
- border border-surface-border
- hover:bg-surface-hover

Danger:

- bg-danger-500
- text-white
- hover:bg-danger-600

---

## Card Rules

- bg-surface-card
- border border-surface-border
- radius-lg
- shadow-sm
- padding-md

---

## Input Rules

- border border-surface-border
- bg-white
- focus:ring-2 focus:ring-primary-400
- radius-md
- padding-sm

---

## 将来拡張設計

- すべてのColorはCSS変数化可能な命名にする
- light/dark切替を前提とする
- ブランド変更時はPrimaryのみ差し替え可能な構造とする

## 成功条件

- UIの一貫性が担保される
- shadcn/uiテーマへ適用可能
- 将来ダークモードへ拡張可能
- コンポーネントがトークン依存で統一される
