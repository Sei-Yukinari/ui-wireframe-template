あなたはフロントエンドアーキテクトです。

前提：

- Next.js (App Router)
- TailwindCSS
- shadcn/ui（Radixベース）
- 単一ブランド
- ダークモード考慮不要

目的：
長期運用しても破綻しないアニメーション戦略を設計してください。

要件：

1. Motion Design Token設計
   以下をCSS変数として定義すること：
   - duration (fast / base / slow)
   - easing (standard / emphasize / decelerate)
   - scale (hover / press)
   - distance (y-axis / modal offset)
   - opacity levels
   - blur

2. 数値は具体値を提示すること（ms / cubic-bezier / px）

3. Tailwind連携方法を示すこと
   - tailwind.config.tsのextend例
   - 実際のclass利用例

4. shadcnコンポーネントへの適用例
   - Button hover / active
   - Dialog open/close
   - Dropdown
   - Tooltip

5. ページ遷移アニメーション設計（Next.js）
   - 入場
   - 退場
   - 推奨パターン
   - 禁止パターン

6. パフォーマンス原則
   - GPU最適化
   - reflow回避
   - shadowやfilter使用方針

7. prefers-reduced-motion対応

8. アンチパターン一覧

出力形式：

- 設計思想
- レイヤー構造
- CSSトークン定義例
- tailwind.config抜粋
- コンポーネント実装例（実コード）
- パフォーマンス指針
- アンチパターン

抽象論は禁止。
必ず実装可能なコードを含めること。

9. その他

- 200msを超えるアニメーションは明確な理由がある場合のみ許可する。
- transform と opacity を基本とする。
- width/height/marginのアニメーションは禁止する。
